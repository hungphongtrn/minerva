# Phase 3.1 Plan 03: Bridge Token Integration and Gateway Endpoint Resolution

## Summary

Hardened the `/runs` data-plane bridge path with sandbox-scoped token authentication, bounded runtime recovery, and deterministic fail-fast error handling. Removed global token fallback and synthetic URL construction in favor of repository-backed authoritative endpoints with 30-second grace-period token rotation support.

## Key Deliverables

### 1. Sandbox-Scoped Token Bundle Authentication

**What changed:**
- Added `BridgeTokenBundle` dataclass with `current`, `previous`, and `previous_expires_at` fields
- Refactored `PicoclawBridgeService` to require token_bundle parameter (fail-closed)
- Updated `RunService` to resolve tokens from `SandboxInstanceRepository.resolve_bridge_tokens()`
- Removed dependency on `settings.PICOCLAW_BRIDGE_TOKEN` global fallback

**Key code locations:**
- `picoclaw_bridge_service.py:67-102`: BridgeTokenBundle with grace period validation
- `run_service.py:823-910`: Token resolution from sandbox metadata

### 2. Authoritative Endpoint Resolution

**What changed:**
- Replaced `_get_sandbox_url()` synthetic URL construction with `_get_authoritative_sandbox_url()`
- Only accepts explicitly provisioned `gateway_url` from sandbox metadata
- Fail-closed: returns `None` when no authoritative URL available
- Removed fallback to `http://sandbox-{id[:8]}:18790` pattern

**Key code locations:**
- `run_service.py:911-938`: Authoritative URL resolution (no fabrication)

### 3. Bounded Runtime Recovery

**What changed:**
- Implemented 3-attempt bounded recovery loop in `_execute_via_bridge()`
- Added `_is_recoverable_bridge_error()` to classify retry eligibility
- Recoverable: HEALTH_CHECK_FAILED, TIMEOUT, TRANSPORT_ERROR
- Non-recoverable: AUTH_FAILED, UPSTREAM_ERROR, MALFORMED_RESPONSE (fail-fast)
- Added `_recover_routing_target()` to force reprovision on recovery

**Key code locations:**
- `run_service.py:823-908`: Bounded recovery with retry logic
- `run_service.py:988-1019`: Recoverable error classification

### 4. Typed Error Remediation

**What changed:**
- Updated `_map_routing_error()` in `runs.py` with bridge-specific error types
- Added 503 for BRIDGE_AUTH_FAILED, BRIDGE_TRANSPORT_ERROR
- Added 504 for BRIDGE_TIMEOUT
- Added 502 for BRIDGE_UPSTREAM_ERROR, BRIDGE_MALFORMED_RESPONSE
- All errors include actionable remediation guidance

**Key code locations:**
- `runs.py:415-474`: Bridge error mapping with remediation

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Grace period: 30 seconds | Matches Daytona reprovision cutover window |
| Max recovery: 3 attempts | Balances resilience against indefinite hangs |
| Fail-closed on missing token | Prevents accidental unauthenticated requests |
| No synthetic URL construction | Eliminates class of endpoint resolution bugs |
| Repository-based token resolution | Single source of truth for sandbox state |

## Migration Notes

- Existing sandboxes without `gateway_url` will fail-fast with typed error
- Existing sandboxes without `bridge_auth_token` will fail authentication
- Clients continue using `/runs` endpoint (no API contract change)
- No database migration required (uses existing 03.1-01 schema)

## Security Improvements

1. **Per-sandbox token isolation**: Each sandbox has unique auth token
2. **Token rotation grace period**: In-flight requests complete during cutover
3. **Fail-closed authentication**: Missing token = rejected request
4. **No token exposure in logs**: Token bundle passed programmatically
5. **Authoritative endpoints only**: Cannot be tricked into connecting to wrong sandbox
