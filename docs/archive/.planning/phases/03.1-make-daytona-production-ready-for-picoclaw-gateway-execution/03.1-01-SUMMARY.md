## What Was Delivered

Added the persistence contract needed for Daytona production-hardening: authoritative endpoint storage, per-sandbox bridge token rotation, and readiness/hydration status tracking.

### Key Deliverables

1. **Alembic Migration 0006** (`src/db/migrations/versions/0006_add_sandbox_bridge_auth_and_readiness_columns.py`)
   - Bridge auth token fields: `bridge_auth_token`, `bridge_auth_token_prev`, `bridge_auth_token_prev_expires_at`
   - Readiness gate: `identity_ready` (Boolean, default False)
   - Hydration tracking: `hydration_status`, `hydration_retry_count`, `hydration_last_error`
   - Idempotent operations with PostgreSQL enum creation

2. **Updated ORM** (`src/db/models.py`)
   - Added `SandboxHydrationStatus` enum (pending, in_progress, completed, degraded, failed)
   - Added bridge token rotation columns to `SandboxInstance`
   - Added readiness and hydration tracking columns

3. **Repository APIs** (`src/db/repositories/sandbox_instance_repository.py`)
   - `rotate_bridge_token(sandbox_id, new_token, grace_seconds=30)`: Moves current token to previous slot with expiry
   - `resolve_bridge_tokens(sandbox_id)`: Returns current + valid previous tokens
   - `set_gateway_url_authoritative(sandbox_id, gateway_url)`: Sets canonical endpoint
   - `set_identity_ready(sandbox_id, ready)`: Controls request acceptance gate
   - `set_hydration_status(sandbox_id, status, last_error)`: Tracks checkpoint restore state
   - `increment_hydration_retry(sandbox_id, error)`: Tracks retry budget consumption
   - `list_identity_not_ready()` / `list_hydration_degraded()`: Diagnostic queries

4. **Behavior Tests** (`src/tests/services/test_sandbox_instance_repository.py`)
   - 36 tests covering token rotation grace overlap, gateway authority, identity readiness, hydration persistence
   - Fail-closed assertions for missing tokens/gateway
   - State persistence tests across unrelated updates

## Must-Haves Verified

✅ **Each sandbox record stores an authoritative gateway endpoint plus sandbox-scoped bridge auth token data required by orchestrated bridge execution.**
- `gateway_url` column for authoritative endpoint
- `bridge_auth_token` column for per-sandbox token
- Repository methods for token rotation and gateway authority

✅ **Bridge token rotation preserves a 30-second grace overlap so in-flight requests do not fail during reprovision cutover.**
- `rotate_bridge_token()` moves current to previous with configurable grace period
- `resolve_bridge_tokens()` returns previous token only while within grace window
- Tests verify grace period expiration excludes old token

✅ **Readiness and hydration degradation state is queryable from sandbox persistence data for routing diagnostics.**
- `identity_ready` Boolean field gates request acceptance
- `hydration_status` enum tracks checkpoint restore progress
- `hydration_retry_count` and `hydration_last_error` track failure context
- Query methods `list_identity_not_ready()` and `list_hydration_degraded()` for diagnostics

## Technical Notes

- Migration uses idempotent operations (checks column existence before add/drop)
- PostgreSQL enum created explicitly before column addition
- Repository follows fail-closed pattern (identity_ready defaults to False, missing tokens return None)
- Token rotation maintains only one previous token (not a chain) to simplify reasoning
- Grace period is configurable but defaults to 30 seconds per specification
