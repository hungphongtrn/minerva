## Decisions Made

1. **Image.from_dockerfile() over manual build steps** - Picoclaw Dockerfile is self-contained, so use from_dockerfile() which handles build context automatically
2. **Shallow clone (--depth 1)** - Faster repository cloning since we only need the specified ref, not full history
3. **Log streaming callback pattern** - Service accepts optional on_logs callback for real-time output, making it testable without CLI coupling
4. **CLI args override env vars** - When both are present, CLI args take precedence for explicit operator control
5. **Structured error remediation** - Daytona permission and auth errors include specific guidance (e.g., "write:snapshots scope required")

## User Setup Required

**Required environment variables for snapshot build:**

| Variable | Description | Example |
|----------|-------------|---------|
| `PICOCLAW_REPO_URL` | Git repository URL for Picoclaw source | `https://github.com/your-org/picoclaw` |
| `PICOCLAW_REPO_REF` | Git ref (branch/tag/sha) | `main`, `v1.0.0`, or commit SHA |
| `DAYTONA_PICOCLAW_SNAPSHOT_NAME` | Target snapshot name | `picoclaw-base` |

**Verification:**
```bash
# Build snapshot
uv run minerva snapshot build

# Or with explicit args
uv run minerva snapshot build \
  --repo-url https://github.com/your-org/picoclaw \
  --ref main \
  --name picoclaw-base

# Verify in .env
DAYTONA_PICOCLAW_SNAPSHOT_NAME=picoclaw-base
```

**Daytona API requirements:**
- API key with `write:snapshots` scope
- Set `DAYTONA_API_KEY` environment variable
- For self-hosted Daytona, also set `DAYTONA_API_URL`
