  - Ensures a durable workspace exists for that user (reuse existing workspace-lifecycle patterns).

  - Async per-user lock keyed by `X-User-ID`.
  - Add in-process idempotency state keyed by `(X-User-ID, X-Idempotency-Key)`:
    - If a request with the same key is in-flight, wait for completion and replay the cached terminal event(s) to the caller.
    - If completed recently, replay cached events immediately.
    - Apply TTL + max entries per user to prevent unbounded memory growth.
- Update `RunService` to support session continuity:
  - Add optional `session_id` parameter to `execute_with_routing`.
  - Update session key generation to use `minerva:{workspace_id}:{pack_scope}:{session_id}` when provided.
  - Preserve current behavior for non-OSS callers (no session_id -> unique per call).

- Add OSS `POST /runs` route (root-level, no `/api/v1` prefix) using `EventSourceResponse`:
  - Uses `resolve_external_principal` + `OssUserQueue`.
  - Reads optional headers:
    - `X-Session-ID` for session continuity.
    - `X-Idempotency-Key` for request idempotency (required for idempotency behavior; if omitted, proceed non-idempotently).
  - Emits provisioning status events during cold start (at minimum: `provisioning` with step labels), then forwards final agent output as a `message` event.
  - MUST call `RunService.execute_with_routing(...)` (explicitly; do not bypass the routing layer).
  - One auto-retry for transient failures:
    - If the first attempt fails with a retryable Daytona/bridge/network transient error, retry exactly once, then emit a typed `error` event and close.
  - Idempotency behavior:
    - If a request arrives with an `X-Idempotency-Key` that is already in progress or completed, return a stream that replays the cached result instead of re-executing.
  - Never leaks internal sandbox URLs, tokens, stack traces, or DB IDs in SSE payloads.
- Tests:
  - Assert SSE content type and that events arrive in order: queued/provisioning*/running/message/completed (happy path).
  - Assert provisioning events appear for first request (mock Daytona/routing where needed).
  - Assert duplicate request with same `X-Idempotency-Key` does not call execution twice and replays the same terminal event.
  </action>

- Exactly one transient retry occurs, then a typed error if still failing.
