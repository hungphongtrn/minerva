## Decisions Made

1. **/health vs /ready separation**: /health is for monitoring (always 200), /ready is for k8s readiness probes (503 on failures)
2. **Fail-closed readiness**: Any blocking dependency failure returns 503 with remediation
3. **Snapshot gate**: /ready fails if configured snapshot doesn't exist, with "run minerva snapshot build" remediation
4. **No auth for operator endpoints**: /health, /ready, /metrics are accessible without authentication for k8s operator use
5. **Root-level mounting**: OSS endpoints at root, API at /api/v1 (additive pattern)
