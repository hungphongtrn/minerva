  <action>
Create `src/scripts/picoclaw_gateway_audit.py` as a small, script-style module (similar shape to `src/scripts/daytona_base_image_preflight.py`) that can be used in both tests and manual audits.

Required behavior:
- Provide a `PicoclawGatewayAuditor` class with an async `run()` that returns a structured result dataclass (JSON-serializable via `to_dict()`).
- Support 2 audit entrypoints:
  1) `audit_existing_gateway(gateway_url, auth_token, message, sender_id, session_id)` for probing a known gateway.
  2) `audit_daytona_sandbox(...)` that provisions a disposable Daytona sandbox using `DaytonaSandboxProvider.provision_sandbox()` with a `SandboxConfig` containing a generated `runtime_bridge_config.bridge.auth_token`, then runs the same probes, then always stops the sandbox via `DaytonaSandboxProvider.stop_sandbox()` in a `finally` block.

Evidence captured in the result:
- `health`: HTTP status + parsed JSON (best-effort) for `GET /health`
- `execute`: HTTP status + parsed JSON (best-effort) for `POST /bridge/execute` performed via `PicoclawBridgeService.execute()` (use a deterministic `session_key` like `minerva:audit:{uuid}`)
- `streaming_probe`: probe a fixed list of candidate paths (documented in result) and record status codes/connection errors; do not assume any exist.
- `continuity_wiring`: echo the `sender_id` and `session_id` used, and confirm the payload forwarded them (record the transformed request dict used by the bridge service).

Implementation constraints:
- Do not add new dependencies.
- Do not attempt to "fix" Picoclaw or change gateway behavior; this is evidence-only.
- Keep output deterministic: no timestamps in markdown; sandbox IDs may vary but should be confined to the JSON result.

CLI:
- Add `main()` with argparse supporting `--daytona` (bool), `--gateway-url`, `--auth-token`, `--message`, `--sender-id`, `--session-id`, `--json`.
- Print human-readable summary by default; when `--json` print only JSON.
  </action>

  </verify>
  <done>
Audit runner provisions/stops Daytona sandboxes safely (finally cleanup) and can probe Picoclaw `/health` and `/bridge/execute` with captured evidence output.
