<objective>
Plumb `external_user_id` end-to-end from OSS principal through routing/orchestration and into sandbox persistence and provider provisioning.

Purpose: Multi-user OSS runtime requires 1 user -> 1 sandbox, and multi-user means multi-sandbox. Without external_user_id in routing, all end-users collide into one sandbox per workspace.

Output: Repository queries keyed by (workspace_id, external_user_id), orchestrator passes external_user_id into SandboxConfig, and RunService/Lifecycle propagate external_user_id from OSS principal.
</objective>

     - If workspace_id cannot be parsed as UUID or workspace is missing, return a failed RunRoutingResult with `WORKSPACE_RESOLUTION_FAILED`.
   - Keep existing behavior for API-key principals and guests.
