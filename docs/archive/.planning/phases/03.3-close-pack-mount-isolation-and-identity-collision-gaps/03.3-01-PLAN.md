<objective>
Decouple end-user identity from the developer `users` table and establish workspace-scoped external identity resolution for multi-tenant safety.

Purpose: The current `resolve_external_principal` creates `User` records for end-user X-User-ID values, enabling privilege escalation (end-user ID matching developer email) and creating per-user workspaces (end-users can't access developer's packs). This plan fixes both by introducing a separate `external_identities` table and resolving all end-users to the developer's workspace.

Output: ExternalIdentity model, Alembic migration, MINERVA_WORKSPACE_ID/GUEST_ID settings, rewritten identity resolution, preflight workspace validation.
</objective>

1. **Completely rewrite `src/api/dependencies/external_identity.py`:**
   The current implementation creates `User` records from X-User-ID values (the identity collision bug). Replace entirely:

   - Update `ExternalPrincipal` NamedTuple:
     - `workspace_id: str` — from MINERVA_WORKSPACE_ID (the developer's workspace)
     - `external_user_id: str` — raw X-User-ID value
     - `is_guest: bool = False` — True when X-User-ID matches GUEST_ID
     - Remove `user_id` field (no longer relevant — end-users don't have User records)
     - Keep `is_active: bool = True`

   - Rewrite `resolve_external_principal`:
     a. Read `settings.MINERVA_WORKSPACE_ID` — if not set, return 403 with detail: "MINERVA_WORKSPACE_ID not configured. Run `minerva register` to get your workspace ID, then set MINERVA_WORKSPACE_ID in your environment."
     b. Require X-User-ID header (400 if missing, same as before)
     c. Length validation (400 if > 255 chars, same as before)
     d. Guest check: if `settings.GUEST_ID` is set and `x_user_id == settings.GUEST_ID`, return ExternalPrincipal with `is_guest=True`. No DB operations for guests.
     e. For non-guest: upsert into `external_identities` table:
        - Query `ExternalIdentity` where `workspace_id == UUID(settings.MINERVA_WORKSPACE_ID)` AND `external_user_id == x_user_id`
        - If not found, create new `ExternalIdentity(workspace_id=UUID(settings.MINERVA_WORKSPACE_ID), external_user_id=x_user_id)`
        - `db.add()` + `db.flush()`
     f. Return `ExternalPrincipal(workspace_id=settings.MINERVA_WORKSPACE_ID, external_user_id=x_user_id, is_guest=False)`

   - CRITICAL: Do NOT import or query the `User` model. End-users never touch the `users` table.
   - Import `ExternalIdentity` from `src.db.models` and `settings` from `src.config.settings`

2. **Add preflight validation checks to `src/services/preflight_service.py`:**
   Add two new checks to `run_all_checks()` method:

   a. `_check_workspace_configured()` (BLOCKING severity):
      - Check `settings.MINERVA_WORKSPACE_ID` is set and non-empty
      - If set, verify the workspace UUID exists in the database (query Workspace table by id)
      - If workspace exists, verify it has at least one registered AgentPack (query AgentPack where workspace_id matches AND is_active=True)
      - If workspace has no packs: FAIL with remediation "Workspace has no registered agent packs. Run `minerva register` first."
      - If workspace has multiple packs: WARNING "OSS mode supports one agent pack per workspace. Using first registered pack."
      - If MINERVA_WORKSPACE_ID is not set: FAIL with remediation "Set MINERVA_WORKSPACE_ID. Run `minerva register` to get your workspace ID."

   b. Register the new check in `run_all_checks()` list

   Note: The preflight DB checks require an engine. Use the existing `_get_db_engine()` pattern with `from sqlalchemy import text` for the workspace/pack queries, or import models directly.
