## Decisions Made

1. **VolumeMount read_only via additional_properties**: The Daytona SDK's VolumeMount class uses Pydantic and accepts extensible properties through the `additional_properties` dict. This is the idiomatic way to pass read-only semantics to the underlying API.

2. **Metadata contract for parity**: Both providers expose identical metadata keys (`pack_mount_path`, `pack_mount_read_only`) enabling tests to verify behavioral parity without requiring real infrastructure.

3. **Fail-closed guard in local_compose**: While local_compose is a simulation, adding a guard that raises `SandboxConfigurationError` if dynamic paths would be placed under `/workspace/pack` keeps the simulation honest and catches contract violations early.
