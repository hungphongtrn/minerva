## Decisions Made

1. **Option A (preferred)**: Changed `_generate_runtime_bridge_config()` signature to accept `bridge_auth_token: str` parameter rather than generating internally. This makes the dependency explicit and prevents accidental internal generation.

2. **Single token generation point**: Token is generated in `_provision_sandbox()` before any config operations, then passed to both functions that need it. This ensures consistency by construction.

3. **Preserved grace period**: The existing `rotate_bridge_token()` with `grace_seconds=30` is maintained - no changes needed to the rotation logic itself.
