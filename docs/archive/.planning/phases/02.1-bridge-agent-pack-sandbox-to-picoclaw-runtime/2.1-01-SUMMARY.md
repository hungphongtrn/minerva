# Phase 2.1 Plan 01: Picoclaw Bridge Service Summary

## One-Liner

Implemented PicoclawBridgeService with health-first bearer-authenticated execution, typed error mapping, and deterministic retry/timeout behavior.

## What Was Built

### 1. PicoclawBridgeService (`src/services/picoclaw_bridge_service.py`)

**Core capabilities:**
- **Health-first execution**: Always polls `/health` before any execute request
- **Bearer token authentication**: Token attached to both health and execute calls
- **Typed error categories**: 
  - `HEALTH_CHECK_FAILED` - Sandbox unhealthy
  - `AUTH_FAILED` - Authentication rejected
  - `TIMEOUT` - Request exceeded timeout
  - `TRANSPORT_ERROR` - Network/connection issues
  - `UPSTREAM_ERROR` - Non-2xx from Picoclaw
  - `MALFORMED_RESPONSE` - Invalid JSON response
- **Fail-closed**: Never attempts execution when health/auth checks fail
- **Deterministic retry**: Bounded retries with exponential backoff
- **Remediation guidance**: All errors include actionable remediation strings

**Key interfaces:**
```python
class PicoclawBridgeService:
    async def check_health(self, sandbox_url: str) -> HealthStatus
    async def poll_health(self, sandbox_url: str) -> HealthStatus
    async def execute(
        self,
        sandbox_url: str,
        message: str,
        session_key: str,
        workspace_id: Optional[str] = None,
        agent_pack_id: Optional[str] = None,
        run_id: Optional[str] = None,
    ) -> BridgeResult
```

### 2. Bridge Configuration (`src/config/settings.py`)

Added settings:
- `PICOCLAW_BRIDGE_TOKEN`: Bearer token for gateway auth
- `PICOCLAW_BRIDGE`: Dict with nested config:
  - `HEALTH_TIMEOUT`: seconds (default: 10)
  - `HEALTH_RETRIES`: count (default: 3)
  - `HEALTH_BACKOFF`: seconds (default: 1.0)
  - `EXECUTE_TIMEOUT`: seconds (default: 300)
  - `EXECUTE_RETRIES`: count (default: 0)
