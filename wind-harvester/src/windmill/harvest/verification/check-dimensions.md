# Verification Dimensions

Verify every implemented bean against three dimensions.

## Completeness

- all requirements are checked off
- files from the plan exist
- planned tests exist

## Correctness

- code builds or compiles
- tests pass
- behavior matches requirement intent
- expected error handling exists

## Coherence

- code follows the design doc and project conventions
- no leftover debugging artifacts remain
- docs are updated when the plan required them
