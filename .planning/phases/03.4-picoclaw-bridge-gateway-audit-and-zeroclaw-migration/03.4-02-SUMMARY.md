## Decisions Made

- **Structural invariants only**: The test asserts that the result has the expected structure (health, execute, streaming_probe, continuity_wiring evidence), not that the audit passes. This allows the same test to collect evidence for both PASS and FAIL verdicts.
- **Graceful skip**: When DAYTONA_API_KEY or LLM_API_KEY are not set, the test skips with a clear message explaining how to configure the environment. This allows CI to run without failing when credentials aren't available.
- **Soak window definition**: 24 hours minimum duration, at least 100 production runs, less than 1% error rate, and zero authentication failures required before considering migration safe.
- **Template approach**: The audit report is created as a framework/template that gets populated with actual results after live execution. This separates the audit infrastructure from the evidence collection.
