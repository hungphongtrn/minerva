## Gaps Summary

### Gap 1: Deprecated Method Has Runtime Error

**Issue:** `RunService._is_recoverable_bridge_error()` references `BridgeErrorType` which was defined in the deleted `picoclaw_bridge_service.py` module.

**Location:** src/services/run_service.py lines 1318-1320

**Impact:** If this deprecated method is called, it will raise `NameError: name 'BridgeErrorType' is not defined`

**Fix:** Update references from `BridgeErrorType` to `GatewayErrorType`, or remove the deprecated method entirely since `_is_recoverable_gateway_error()` exists as the replacement.

### Gap 2: Legacy Test File Imports Deleted Module

**Issue:** `test_phase2_1_bridge_execution.py` has 6 import statements referencing `src.services.picoclaw_bridge_service` which no longer exists.

**Location:** src/tests/integration/test_phase2_1_bridge_execution.py lines 129, 146, 163, 403, 428, 463

**Impact:** If this test file is executed, it will raise `ImportError: cannot import name 'BridgeError' from 'src.services.picoclaw_bridge_service'` (file not found)

**Fix:** Update the test file to import from `zeroclaw_gateway_service` instead, or mark the entire test file as deprecated/obsolete.
