## Files Created/Modified

- `src/api/oss/routes/runs.py` - Integrated upstream event extraction and emission in `_execute_run_with_events()`
- `src/tests/services/test_event_mapping.py` - 29 comprehensive unit tests for the mapping functionality

## Decisions Made

1. **Upstream events location**: Events stored in `gateway_result.output.events` field, consistent with ZeroClaw Gateway API structure
2. **Unknown event handling**: Mapping helper returns `None` for unknown types, allowing graceful ignore without breaking the stream
3. **Double-terminal prevention**: If upstream emits `completed` or `failed`, `/runs` skips its own terminal event to avoid duplicates
4. **Non-streaming fallback**: When no upstream events present, falls back to existing best-effort final message extraction
