  <action>
Implement a mapping helper in `src/services/oss_sse_events.py` that converts upstream ZeroClaw runtime events into `OssSseEvent` instances.

Requirements:
1) Provide a helper that can be used from the `/runs` generator, e.g.:
   - `def map_zeroclaw_event_to_oss_event(builder: OssSseEventBuilder, upstream: dict) -> Optional[OssSseEvent]: ...`
2) Support these upstream event type strings (best-effort; ignore unknown types):
   - message: `message.delta`, `message`
   - tool: `tool.call`, `tool.result`
   - ui/state: `ui.patch`, `state.update`
   - error/terminal: `error`, `completed`, `failed`
3) Map to Minerva `OssEventType`:
   - message.* -> MESSAGE
   - tool.call -> TOOL_CALL
   - tool.result -> TOOL_RESULT
   - ui.patch -> UI_PATCH
   - state.update -> STATE_UPDATE
   - error -> ERROR
   - completed -> COMPLETED
   - failed -> FAILED
4) Preserve payload in `event.data` and never raise on missing keys.
5) Do not introduce any new external dependencies.
