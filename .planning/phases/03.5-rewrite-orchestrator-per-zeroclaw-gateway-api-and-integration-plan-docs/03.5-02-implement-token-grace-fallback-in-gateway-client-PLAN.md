  <action>
Update `ZeroclawGatewayService` so token rotation with grace period works in real HTTP flows.

Required behavior (keep current public method signatures unless strictly necessary):
1) For authenticated specs (`auth.mode == "bearer"`), all gateway calls include Authorization headers.
2) On 401/403 from `/health` or `/webhook`, retry exactly once using `GatewayTokenBundle.previous` IF AND ONLY IF `GatewayTokenBundle.is_grace_token_valid()` is True.
3) If the grace token attempt also yields 401/403, return `GatewayErrorType.AUTH_FAILED` (fail closed).
4) Keep health-first semantics: `execute()` must still short-circuit on unhealthy/unauthorized health.
5) Keep execute retry semantics separate from auth fallback: do not require `execute_retries > 0` for the grace-token retry to happen.
6) Do not leak tokens in returned error messages.

Implementation hint:
- Add a small internal helper that runs `attempt=0` then `attempt=1` (grace) specifically for auth failures.
- Ensure compatibility fallback between `/webhook` and `/execute` remains intact.
