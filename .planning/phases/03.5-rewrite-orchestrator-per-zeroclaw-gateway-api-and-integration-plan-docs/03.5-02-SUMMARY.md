## Decisions Made

1. **Auth fallback is independent of execute_retries**: The grace-token retry happens even when `execute_retries=0`, because token rotation is a transient auth issue, not a server error that should be retried.

2. **Only one grace token retry per request**: Added `auth_fallback_attempted` flag to prevent duplicate grace retries within a single request, avoiding potential loops.

3. **Health-first semantics preserved**: If the health check fails with 401/403 and grace token is invalid/expired, execute is blocked with AUTH_FAILED. This maintains the fail-closed behavior.

4. **Datetime consistency**: Used `datetime.utcnow()` consistently across implementation and tests to avoid timezone mismatches (with deprecation warning acknowledgment for future migration to timezone-aware datetimes).

## Issues Encountered

1. **Test mock updates required**: The existing tests used `httpx.AsyncClient.get` and `httpx.AsyncClient.post` mocks, but the new implementation uses `client.request()` method. Updated all 37 existing tests to use `request` mocking pattern instead.

2. **Datetime timezone mismatch**: Initial test failures due to mixing `datetime.now()` (local time) with `datetime.utcnow()` (UTC). Fixed by using `datetime.utcnow()` consistently in test fixtures.
