## Decisions Made

1. **X-User-ID mapping**: Treat as opaque string mapped to users.email for get-or-create semantics
2. **Idempotency scope**: In-process only with TTL; multi-replica deployments need sticky routing
3. **Session key format**: `minerva:{workspace_id}:{pack_scope}:{session_id}` when session_id provided
4. **Cold start detection**: Check for existing sandboxes in workspace
5. **Error sanitization**: Regex-based removal of tokens, URLs, UUIDs, stack traces
