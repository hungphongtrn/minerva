  <name>task 1: Add explicit "exists -> reuse" semantics to snapshot build</name>
  <files>src/services/daytona_snapshot_build_service.py</files>
  <action>
- Update `DaytonaSnapshotBuildService.build_snapshot()` to:
  - Call `await daytona.snapshot.get(self.snapshot_name)` first.
  - If it returns successfully, treat that as "reused" and return success without calling `snapshot.create`.
  - If it fails with a "not found" DaytonaError, proceed to build+create.
  - If it fails with an auth/permission error, fail closed with remediation (do not attempt create).
- Extend `SnapshotBuildResult` with a boolean field (e.g., `reused: bool`) so CLI/tests can assert behavior.
- Keep log output deterministic: on reuse, emit a short message like "Snapshot already exists; reusing".
  </action>
  <verify>
uv run pytest -q src/tests/services/test_phase3_2_snapshot_build.py
  </verify>
  <done>
- When mocked `snapshot.get(name)` succeeds, `snapshot.create(...)` is not called and result indicates reuse.
- When `snapshot.get(name)` is "not found", the service proceeds to create.
  </done>
</task>
