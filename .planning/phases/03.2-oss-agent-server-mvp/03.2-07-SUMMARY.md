## Decisions Made

1. **Use get-before-create pattern:** Check `daytona.snapshot.get()` before attempting `daytona.snapshot.create()` to enable idempotent operation.

2. **Add explicit `reused` boolean flag:** Extending `SnapshotBuildResult` with `reused: bool = False` provides explicit observability for CLI and tests to discriminate between "created" and "reused" outcomes.

3. **Fail closed on auth errors:** When `snapshot.get()` fails with auth/permission errors (not "not found"), the service fails immediately without attempting `snapshot.create()`. This prevents confusing error messages and unnecessary operations when the caller lacks proper permissions.

4. **Multiple "not found" patterns:** Error string matching includes "not found", "404", "does not exist", and "no such snapshot" to handle different Daytona SDK error messages.
