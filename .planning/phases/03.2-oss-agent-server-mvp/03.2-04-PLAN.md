  <action>
- Add `resolve_external_principal` that:
  - Requires `X-User-ID` header; returns 400 if missing.
  - Uses DB to get-or-create `User` by `users.email == X-User-ID` (treat as opaque string; no format validation; only enforce DB length bounds).
  - Ensures a durable workspace exists for that user (reuse existing workspace-lifecycle patterns).
  - Returns a `Principal` (from `src.identity.key_material`) with `workspace_id` and `user_id` set to UUID strings.
  - Never creates API keys (gateway owns auth).
- Add `OssUserQueue`:
  - Async per-user lock keyed by `X-User-ID`.
  - Exposes `async with queue.acquire(user_id)` used by OSS /runs so concurrent requests wait instead of returning 409.
- Update `RunService` to support session continuity:
  - Add optional `session_id` parameter to `execute_with_routing`.
  - Update session key generation to use `minerva:{workspace_id}:{pack_scope}:{session_id}` when provided.
  - Preserve current behavior for non-OSS callers (no session_id -> unique per call).
  </action>
  <verify>
uv run pytest -q src/tests/services/test_phase3_2_user_queue.py
  </verify>
  <done>
- Two concurrent OSS runs for same `X-User-ID` serialize without 409s.
- Same `X-User-ID` + same `X-Session-ID` maps to the same Picoclaw `chat_id` / session key.
  </done>
</task>

- Create a typed SSE event helper in `src/services/oss_sse_events.py`:
  - Envelope fields (OpenCode discretion, but must be stable): `id`, `type`, `ts`, `run_id`, `data`.
  - Emits lifecycle events: `queued`, `running`, `completed`, `failed`.
  - Emits domain events: `message`, `tool_call`, `tool_result`, `ui_patch`, `state_update`, `error`.
- Implement OSS `POST /runs` as an `EventSourceResponse` stream:
  - Uses `resolve_external_principal` + `OssUserQueue`.
  - Emits provisioning status events during cold start (at minimum: `provisioning` with step labels), then forwards final agent output as a `message` event.
  - On errors, emits a typed `error` event with category and sanitized message, then closes stream.
  - Never leaks internal sandbox URLs, tokens, stack traces, or DB IDs in SSE payloads.
- Tests:
  - Assert SSE content type and that events arrive in order: queued/provisioning*/running/message/completed (happy path).
  - Assert provisioning events appear for first request (mock Daytona/routing where needed).
