<objective>
Implement mount isolation in the Daytona provider to separate static pack files from dynamic runtime data, and update the base provider contract for per-user sandbox routing.

Purpose: Currently both pack identity files and Picoclaw runtime data (memory/, sessions/, cron/) share `/workspace/pack`, causing cross-sandbox session leakage and contaminating the immutable volume-per-digest contract. This plan creates proper isolation: pack volume read-only at `/workspace/pack`, dynamic workspace at `/home/daytona/workspace`, with symlinks bridging identity files.

Output: Updated base SandboxConfig, Daytona provider with symlink-based mount isolation, updated config generation, and workspace-path identity verification.
</objective>

2. **Add `_create_workspace_symlinks` method:**
   ```python
   async def _create_workspace_symlinks(self, sandbox) -> None:
       """Create workspace directory and symlink identity files from pack volume.
       
       Creates /home/daytona/workspace/ and symlinks:
       - AGENT.md, SOUL.md, IDENTITY.md (files)
       - skills/ (directory)
       from /workspace/pack/ into the workspace directory.
       """
       from src.infrastructure.sandbox.providers.base import (
           PACK_MOUNT_PATH, WORKSPACE_PATH, IDENTITY_FILES, IDENTITY_DIRS
       )
       
       # Create workspace directory
       await sandbox.process.exec(f"mkdir -p {WORKSPACE_PATH}")
       
       # Symlink identity files
       for f in IDENTITY_FILES:
           await sandbox.process.exec(
               f"ln -sf {PACK_MOUNT_PATH}/{f} {WORKSPACE_PATH}/{f}"
           )
       
       # Symlink identity directories
       for d in IDENTITY_DIRS:
           await sandbox.process.exec(
               f"ln -sf {PACK_MOUNT_PATH}/{d} {WORKSPACE_PATH}/{d}"
           )
   ```
   Use `ln -sf` (force) to handle re-provisioning scenarios.
