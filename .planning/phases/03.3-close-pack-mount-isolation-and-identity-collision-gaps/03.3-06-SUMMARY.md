## Decisions Made

1. **Repository filtering**: Only filter by external_user_id when explicitly provided (None means include all). This preserves backwards compatibility for API-key based sandboxes.

2. **OSS principal detection**: Check for both workspace_id and external_user_id attributes on principal. This distinguishes OSS ExternalPrincipal from API-key principals.

3. **Fail-closed validation**: OSS path validates workspace_id is valid UUID and workspace exists. Returns WORKSPACE_RESOLUTION_FAILED if either check fails.

4. **Common routing extraction**: Extracted routing result processing into _process_routing_target() helper to avoid code duplication between OSS and standard paths.

5. **Bug fix during implementation**: Fixed list_identity_not_ready() which used incorrect SQLAlchemy syntax (`not SandboxInstance.identity_ready` instead of `SandboxInstance.identity_ready.is_(False)`).
